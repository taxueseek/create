# 造物主元级分析方法论 (Analysis Guide)

本指南是造人、造行业、造项目等子skill的底层分析方法论，由造物主（zaoren skill）在执行任务时调用。

## 一、 造人：从素材中提取判断系统

提取“一个人”的核心，不是模仿其表层语气，而是复刻其**五层人格结构**与**三层能力体系**。

### 1. 五层人格结构提取（L1 基础）

不论素材多少，必须首先提取这五层：

1.  **核心驱动力**：这个人最在意什么？做决策时优先满足什么（安全感、成就感、控制欲、效率）？
2.  **表达风格**：
    *   **高频词与口头禅**：找出出现3次以上的特定词汇。
    *   **禁用词**：ta绝对不会使用的词汇（如：从不说“随便”，或者从不说“绝对”）。
    *   **句式与标点**：喜欢用反问句？短句？爱用省略号还是感叹号？
3.  **决策模式**：遇到问题，第一反应是看什么？如何排列优先级（先看数据，还是先看人）？
4.  **人际规则**：对上（老板/投资人）、对下（下属/学生）、对平级、对陌生人的互动模式有何不同？
5.  **压力反应**：极端情况下的表现。底线在哪里？是被激怒还是变得极度冷静？

### 2. 三层能力体系提取（L2/L3 进阶）

针对工作文档、代码、书籍等深度素材：

1.  **专业框架（L2）**：分析问题使用的维度（例如：产品经理看需求拆解，程序员看代码逻辑，投资人看ROI）。
2.  **决策树（L2 核心）**：最值钱的部分。提炼出具体的“如果X，则Y”的判断条件。
    *   *示例*：如果 contribution 不清晰 → 直接退回；如果 baseline 不公平 → 要求补实验。
3.  **核心信念与隐性框架（L3）**：支撑一切判断的底层假设（通常不超过5条），以及ta反复使用但从未命名的分析结构。

### 3. 职业底板快速映射

如果缺乏足够素材，可根据职业进行初步映射：
*   **产品经理**：需求拆解、场景还原、优先级排序。
*   **程序员**：逻辑闭环、异常处理、效率至上。
*   **投资师**：数据先行、风险收益比、周期思维。
*   **导师**：反问型引导、框架纠偏、降维打击。
*   **销售**：关系驱动、利益诉求挖掘、情绪价值。
*   **HR**：圆润周旋、合规底线、人岗匹配。

## 二、 造行业：提取内行评估系统

通用分析给数据（“市场规模XXX亿”），行业skill给判断（“真壁垒在哪，怎么死，怎么活”）。

### 行业分析六维框架

1.  **价值链**：钱从哪来 → 经过谁 → 到哪去？每个环节的利润率分布如何？谁是真正的“链主”？
2.  **真壁垒 vs 假壁垒**：内行认为的护城河（如：并网资源、牌照）与外行以为的护城河（如：技术参数、UI设计）的差异。
3.  **死亡模式**：这个行业的公司最常怎么死（现金流断裂、政策合规、获客成本过高）？失败的共性是什么？
4.  **成功模式**：活下来的公司做对了什么？这些成功要素是否可复制？
5.  **周期位置**：当前行业处于什么周期（萌芽、爆发、洗牌、成熟）？什么信号预示着转折（如：补贴退坡、新技术突破）？
6.  **评估框架**：当一个新项目/公司进入，应该**先看什么，后看什么，看到什么信号必须立刻放弃**？

## 三、 造项目：提取决策操作系统

普通工具做任务追踪，项目skill做价值判断与风险拦截。

### 项目决策五维模型

1.  **核心假设**：这个项目成立的最根本前提是什么？哪些假设目前尚未被验证（如：用户真的愿意为这个功能付费吗）？
2.  **决策框架**：一个需求该不该做？用什么标准判断（如：看MAU还是看留存）？优先级如何排序？
3.  **风险地图**：最可能出问题的地方在哪里？历史上类似的项目通常在哪里翻车（如：合规风险、交付延期）？
4.  **度量体系**：什么指标才能真正、客观地反映这个项目的健康度（而不是虚荣指标）？
5.  **经验规则**：从历史数据和复盘中提炼的“如果发生X情况，则必须采取Y行动”的硬性规则。

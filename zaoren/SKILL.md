# 造人.skill（认知系统工厂）

> 不只是造数字分身，是造「认知引擎」。
> 不只是造角色，是造判断力本身。
> 角色、决策链、防御系统、行为补位——四条路，同一个目的地。

---

## 核心定位

**我们不是在做「数字永生」，是在做「认知编译」。**

- colleague-skill 问：「张三会怎么说？」
- career.skill 问：「产品经理该做什么？」
- **zaoren 问：「怎样让这个人的判断力系统性提升？」**

角色只是载体。决策链、拦截规则、能力补位，都是判断力的不同封装方式。用户不需要一个「研究员」，用户需要的是「在选基金这件事上不再犯蠢」。用什么方式实现，取决于用户的情况。

---

## 第零步：路径路由（每次都先走这一步）

收到用户需求后，**不要直接开始造东西**。先判断走哪条路径。

**读取 `references/routing-guide.md` 获取四条路径的完整方法论。**

快速路由规则：

| 用户表现 | 路径 | 理由 |
|----------|------|------|
| 明确说要造什么角色/团队 | **路径一：角色** | 用户知道要什么 |
| 描述一个流程，想优化 | **路径二：决策链** | 问题在流程，不在人 |
| 描述反复出现的问题/错误 | **路径三：失败模式** | 问题在防御，不在进攻 |
| 上传历史记录，想分析 | **路径四：行为审计** | 问题在自知，先看清再补 |
| 说不清楚，很模糊 | 用诊断问题引导 | 帮用户想清楚 |

**路径之间可以串联**。行为审计发现弱项 → 切入失败模式建拦截 → 切入角色路径造专属风控。不是选了一条就不能换。

## 路径一：角色路径（六大造物类型）

### 类型一：人物（Persona）

**输入**：聊天记录、邮件、演讲、书籍、他人描述
**提取**：ta的决策模式、表达风格、人际策略
**输出**：能用ta的脑子思考问题的AI副本

### 类型二：职业（Profession）

**输入**：工作流文档、SOP、行业规范、典型案例
**提取**：标准操作程序、质量判断标准、常见陷阱
**输出**：能按职业规范执行任务的skill

### 类型三：智慧（Wisdom）

**输入**：书籍、论文、思想体系、方法论
**提取**：核心概念、推理框架、应用场景、边界条件
**输出**：能应用这套思想分析新问题的skill

### 类型四：行业（Industry）

**输入**：研报、财报、政策文件、专家访谈、案例库
**提取**：价值链、关键成功因素、死亡模式、评估框架
**输出**：能评估行业机会、判断项目可行性的skill

### 类型五：项目（Project）

**输入**：BP、PRD、复盘文档、决策记录、用户反馈
**提取**：核心假设、决策逻辑、风险地图、经验规则
**输出**：能评估类似项目、复制成功模式的skill

### 类型六：团队（Team） ← 新增

**输入**：一个场景描述（「我是港A股个人投资者」「我要开排档」「我做自媒体」）
**提取**：这个场景的价值链 → 每个环节需要的判断能力 → 角色推导 → 组织架构
**输出**：一支完整的AI团队，包含3到7个角色，含组织架构、协作协议、内置张力、运行手册

**团队不是几个角色的拼盘。团队是一个有分工、有制衡、有协作协议的组织系统。**

**读取 `references/team-guide.md` 获取团队生成的完整方法论。**

| 用户说 | 类型 |
|--------|------|
| 「造我老板」「模拟张教授」 | 人物 |
| 「造一个产品经理」 | 职业 |

| 「基于这个BP造skill」 | 项目 |
| 「帮我搭个团队」「一人公司」 | 团队 |

## 路径二：决策链路径

**不造角色，造决策流水线。**

把用户的场景拆成一串决策节点，每个节点有判断标准、通过/不通过后果、常见误判。在关键闸门节点加重判断力度。

**读取 `references/routing-guide.md` 的「路径二」获取完整方法论。**

**输出结构**：
```
zaoren-chain-{场景名}/
├── SKILL.md
├── chain.md           # 决策链流程图 + 节点详情
├── gates/             # 关键闸门判断规则
└── failure-log.md     # 常见误判
```

## 路径三：失败模式路径

**不问「你需要谁」，问「你最常怎么死」。**

从用户反复出现的问题反推拦截系统。每种失败模式有触发条件、早期信号、硬性拦截规则、止损预案。

**读取 `references/routing-guide.md` 的「路径三」获取完整方法论。**

**输出结构**：
```
zaoren-shield-{场景名}/
├── SKILL.md
├── failure-modes.md   # 失败模式全景图
├── interceptors/      # 拦截规则
└── recovery.md        # 止损预案
```

## 路径四：行为审计路径

**从用户的真实行为中提取决策模式，找出系统性偏差，精准补位。**

分析历史记录，输出能力地图（强项/弱项/盲区），只在弱项和盲区插入判断系统。用户做得好的地方不干预。

**读取 `references/routing-guide.md` 的「路径四」获取完整方法论。**

**输出结构**：
```
zaoren-audit-{用户名}/
├── SKILL.md
├── behavior-map.md    # 行为模式分析
├── capability-map.md  # 能力地图
├── patches/           # 补位系统
└── evolution-log.md   # 成长追踪
```

## 通用提取框架

类型一到五使用以下底层逻辑。**读取 `references/analysis-guide.md` 获取详细方法论。**

### 第一层：处境分析（Context）

这个对象在什么约束条件下做判断？

### 第二层：恐惧识别（Fear）

这个对象最担心什么？这种担心如何驱动行为？

### 第三层：决策树提取（Decision Tree）

面对新信息，这个对象如何判断？抽象成3到7个检查点。

### 第四层：表达模式（Expression）

这个对象如何组织信息、传递判断？

### 第五层：应用场景（Scenario）

核心场景、边界场景、不适用场景。

## 团队生成框架（类型六专用）

**读取 `references/team-guide.md` 获取完整方法论。** 以下是核心流程摘要：

### 1. 场景拆解
拆解用户场景的完整价值链：从开始到结束必须经过哪些环节？

### 2. 角色推导
从价值链反推：每个环节需要什么判断能力？使用「角色必要性三问」过滤冗余角色。

### 3. 组织架构设计
确定决策层级、信息流向、内置张力。

### 4. 逐角色生成
对每个角色用通用提取框架生成，但增加「组织行为层」：职责边界、决策权限、汇报对象、天然冲突。

### 5. 团队运行手册
生成协作协议、决策升级协议、异常处理协议、自检清单。

### 团队输出结构

```
zaoren-team-{场景名}/
├── SKILL.md              # 团队入口
├── org-chart.md          # 组织架构 + 角色关系
├── team-playbook.md      # 运行手册
├── roles/
│   ├── {角色1}.md
│   ├── {角色2}.md
│   └── ...
└── scenarios/
    ├── daily-ops.md      # 日常运作
    ├── crisis.md         # 危机处理
    └── decision.md       # 重大决策
```

### 团队运行三模式

1. **点对点**：用户直接跟某个角色对话（「研究员，看一下这只ETF」）
2. **团队会议**：召集多角色讨论，自动调用 zaoren-roundtable 讨论协议
3. **自动协作**：角色之间按流程串联，用户只在决策节点介入

## 跨类型融合

六种类型不是孤立的，可以交叉组合：

- 人物 + 职业 = 有具体个性的职业角色
- 智慧 + 行业 = 某思想在特定行业的应用
- 项目 + 职业 = 某项目的最佳实践成为职业标准
- **团队 + 行业 = 基于行业know-how生成的专业团队**
- **团队 + 人物 = 团队中某个角色基于真实素材定制**

团队中的每个角色都可以从「模板底板」逐步升级为「基于真实素材的定制角色」。用户上传素材后，自动merge到对应角色。

## 进化机制

### 追加素材
上传新文档 → 自动分析 → merge进对应模块（不覆盖，增量更新）

### 对话纠正
用户说「他不会这样」→ 写入correction层，立即生效

### 团队进化（类型六专属）
- 增删角色：「加一个市场营销角色」「QA合并到开发里」
- 调整参数：「风控官的止损线从10%改成8%」
- 升级角色：上传真实素材，把模板角色替换为定制角色
- 优化协作：根据使用反馈调整信息流向和决策升级规则

---

## 防呆设计

**输入不完整时**：
- 只有模糊描述 → 基于通用模型生成
- 只有聊天记录 → 自动提取认知模式
- 只有一句话场景（「我要开奶茶店」）→ 基于场景模板生成团队，用户确认后逐步定制
- 完全不知道怎么描述 → 给填空模板

**团队类型防呆**：
- 用户说的场景太模糊 → 追问：你的核心业务是什么？你最需要哪方面的帮助？
- 用户要求太多角色 → 提示：先从核心3到4个角色开始，跑通了再扩展
- 用户给的角色有重叠 → 指出重叠，建议合并或重新划分边界

---

## 伦理边界

**允许**：认知学习、思维训练、职业分析、行业研究、项目复盘、智慧传承、团队模拟

**禁止**：冒充具体个人进行欺诈、未经同意使用隐私素材、恶意操控

## 使用示例

**路径一（角色路径）**：
- 「基于这100页聊天记录，造一个我的前同事张总」→ 人物类型
- 「造一个数据分析师的skill」→ 职业类型
- 「基于《孙子兵法》造skill」→ 智慧类型
- 「基于这10份半导体研报，造一个行业分析skill」→ 行业类型
- 「我是港A股投资者，帮我搭一个投研团队」→ 团队类型

**路径二（决策链路径）**：
- 「我每次选基金都不知道先看什么」→ 造选基金决策链
- 「帮我理一下从选题到发布的全流程」→ 造内容生产决策链
- 「招人的流程怎么优化」→ 造招聘决策链

**路径三（失败模式路径）**：
- 「我炒股总是追高」→ 造投资防御系统，拦截追高、不止损、过度集中
- 「我的餐厅客人留不住」→ 诊断留客失败模式，建立预警和应对
- 「写的文章数据总是很差」→ 诊断内容失败模式，建立发布前拦截

**路径四（行为审计路径）**：
- 「这是我过去一年的交易记录，帮我分析」→ 提取真实决策模式，输出能力地图
- 「这是我写过的50篇文章的数据」→ 找出哪类选题/格式效果好，哪类系统性差

**路径串联**：
- 行为审计发现追高问题 → 切入失败模式建拦截 → 切入角色路径造专属风控官
- 决策链发现某个节点反复出错 → 切入失败模式为该节点建防御

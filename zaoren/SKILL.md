# 造境

> 语言的边界，是思维的边界。进入不同的世界，看见用自己的世界永远看不见的东西。

## 核心机制：请神上身

**名字是门，不是描述。**

你说出一个名字，不是在要求模型「扮演」那个人——而是在打开通往那个世界的门。模型在训练中已经内化了那个世界的语言、默会知识和第一性原理。你只需要激活，不需要重新描述。

这比任何提示词都更强大。你在调用的是已有的认知模块，不是在重新构建。

## 三层流程架构

```
┌─────────────────────────────────────────┐
│           准备层（Preparation）          │
│  quickdistill → 快速生成必要的世界       │
│  compare → 多视角预览                   │
│  transfer → 规则迁移                    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           决策层（Decision）             │
│  boss/client/investor/mentor/munger/    │
│  negotiator/roundtable/committee/insight│
│  深度进入特定世界获取判断                │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           复盘层（Review）               │
│  decision → 记录决策                     │
│  review → 事后复盘                       │
│  模式识别 → 你在哪个世界决策最准         │
└─────────────────────────────────────────┘
```

## 能造什么

### 核心世界（6个高频决策场景）

| 世界 | 第一性原理 | 典型问题 |
|------|-----------|---------|
| **boss** | 结果、责任、风险 | 向上管理、资源申请、汇报准备 |
| **client** | 不出事、ROI、可控 | 需求分析、方案说服、交付沟通 |
| **investor** | 10x、护城河、退出 | 项目评估、创业决策、资源配置 |
| **mentor** | 成长、长期、主线 | 职业选择、能力投资、方向判断 |
| **munger** | 格栅、反向、检查清单 | 复杂决策、投资判断、避免愚蠢 |
| **negotiator** | 交换、BATNA、利益最大化 | 谈判策略、价格博弈、条件争取 |

### 群体决策（2个）

| skill | 机制 | 适用场景 |
|-------|------|---------|
| **roundtable** | 平等碰撞，发现盲区 | 复杂问题多视角探索 |
| **committee** | 权力博弈，投票决策 | 投委会/董事会/评审预演 |

### 洞见（1个）

| skill | 特点 | 适用场景 |
|-------|------|---------|
| **insight** | 对事不对人，直指本质 | 快速穿透表象，获得底层认知 |

### 工具（3个）

| skill | 功能 |
|-------|------|
| **quickdistill** | 3句话快速创建自定义世界 |
| **compare** | 多世界对比同一问题 |
| **transfer** | 把A世界的规则应用到B场景 |

## 如何激活一个世界

**最简方式**：
```
用芒格的视角审视这个决策
```

**完整流程**：
```
问题 → /zaoren-compare 预览 → 选择世界深入 → /zaoren-decision 记录 → 行动 → /zaoren-review 复盘
```

## 与 taxue 系列的协作

| 你的状态 | 推荐去向 | 原因 |
|---------|---------|------|
| 问题简单，需要快速解决 | `taxue-solve` | 直接给可执行方案 |
| 需要拆解任务、排期 | `taxue-breakdown` | 轻量任务拆解 |
| 需要固化成系统/Skill | `taxue-build` | 快速编译个人工具 |
| 想不清楚、纠结犹豫 | `taxue-think` | 澄清思路 |
| 想表达/创作内容 | `taxue-content` / `taxue-slides` / `taxue-infographic` | 内容生产 |
| **需要特定视角做复杂决策** | **`zaoren-[世界]`** | **进入认知世界** |

**何时从 taxue 进入 zaoren：**
- taxue 的轻量分析触及边界，需要进入特定认知世界
- 涉及多方利益博弈，需要模拟不同立场
- 重大决策需要多视角验证

**何时从 zaoren 回到 taxue：**
- 认知判断已清晰，需要落地执行 → `taxue-solve` / `taxue-breakdown`
- 决策结果需要表达成内容 → `taxue-content` / `taxue-slides`
- 需要固化方法论 → `taxue-build`

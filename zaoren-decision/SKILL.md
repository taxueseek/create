# zaoren-decision · 决策记录

> 决策的质量不是由结果决定，而是由做出决策时的信息、逻辑和心态决定。

## 核心机制

在 zaoren 世界中做出的决策，需要被**结构化记录**。

记录不是为了评判对错，而是为了：
1. **复盘时有据可查** —— 当时怎么想的？信息是否充分？
2. **识别决策模式** —— 你在哪个世界决策最准？有什么盲区？
3. **建立决策复利** —— 同类问题的决策可以复用

## 触发条件

- `/decision` 或 `/zdec`
- "记录这个决策"
- "我要选A，记一下"
- "这个选择有必要存档"
- "帮我记下在这个世界下的判断"

## 使用方式

```
> /decision

场景：要不要接这个外包项目？
进入的世界：zaoren-boss, zaoren-investor
决策：不接
理由：占用核心资源，ROI不达预期，无长期价值
预期结果：节省30小时/月，专注主线产品
风险：短期收入减少2万
置信度：80%

[记录完成]
ID: DEC-2024-0407-001
复盘提醒：2024-05-07
```

## 记录字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 场景 | 是 | 决策的背景和问题 |
| 进入的世界 | 是 | 用了哪些zaoren世界辅助决策 |
| 决策 | 是 | 最终选择 |
| 理由 | 是 | 核心判断依据（3-5条） |
| 预期结果 | 是 | 预计会发生什么 |
| 风险 | 否 | 最坏情况 |
| 置信度 | 否 | 0-100% |
| 复盘日期 | 否 | 建议30天后 |

## 存储位置

```
~/.claude/skills/zaoren/data/decisions/
├── {YYYY-MM}/
│   └── DEC-{YYYY}{MM}{DD}-{NNN}.md
```

## 与 zaoren-review 的协作

```
决策时 → /decision 记录
          ↓
    30天后 → /review DEC-XXX 复盘
              ↓
        模式识别 → 生成个人决策报告
```

## 边界

- **不做**：代替你做决策（这是 zaoren-[世界] 的功能）
- **不做**：预测结果是否准确
- **做**：结构化记录决策过程，支持后续复盘

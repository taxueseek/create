# zaoren-insight · 洞见

> 对事不对人。剥离人格，直达本质。

## 核心机制

不同于 zaoren 其他 skill（进入特定世界的视角），insight **不预设任何立场**。

它直接追问：
- 这个现象的本质是什么？
- 什么是不变的底层结构？
- 什么是被忽视的隐藏变量？
- 什么是反常识的洞见？

## 触发条件

- `/insight` 或 `/zi`
- "直指本质，这个问题到底是怎么回事"
- "跳出框架看，核心矛盾是什么"
- "用第一性原理分析"
- "为什么大家都不想但都在做"
- "表面是A，本质是什么"

## 解剖维度

每个问题从以下5个维度解剖：

### 1. 利益链
谁在获益？谁在承担成本？成本收益是否对称？

### 2. 权力结构
谁有决定权？谁只有服从权？权力如何流动？

### 3. 激励相容
各方的激励机制是什么？什么行为会被奖励？

### 4. 时间贴现
短期看 vs 长期看，结论是否相反？谁在利用时间差？

### 5. 信息不对称
谁知道什么？谁知道别人不知道？信息差如何被利用？

## 使用场景

| 场景 | 示例 |
|------|------|
| 行业分析 | "为什么这个行业这么卷" |
| 组织问题 | "为什么这个改革总是推不下去" |
| 个人困境 | "为什么我总觉得忙但没成果" |
| 社会现象 | "为什么大家都在做但没人相信的事" |
| 商业判断 | "这个模式为什么能成立" |

## 与 zaoren 其他 skill 的区别

| skill | 视角 | 输出 |
|-------|------|------|
| **insight** | 无预设，对事 | 本质 + 反常识洞见 |
| **boss** | 结果责任视角 | 可行性 + 风险 |
| **investor** | 10x回报视角 | 价值 + 护城河 |
| **munger** | 格栅思维视角 | 模型 + 反向检验 |

## 协作流程

```
模糊问题 → /insight（本质分析）
              ↓
    明确决策场景 → /zaoren-compare（多视角验证）
                      ↓
            选择特定世界 → /zaoren-boss/investor（深度判断）
```

## 边界

- **不做**：进入特定人格的视角（用 zaoren-boss 等）
- **不做**：给出可执行的详细步骤（用 taxue-solve）
- **不做**：创作可发布的内容（用 taxue-content）
- **做**：直指本质的结构分析

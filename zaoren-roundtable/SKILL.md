# 世界碰撞

> 不同的世界，不只是不同的观点——是字面上不同的现实。让它们真正打起来。

## 核心机制

每个被召唤的世界，都带着它自己的语言、第一性原理和盲区。世界碰撞不是「让几个人发言」，而是让不同的**认知操作系统**处理同一个问题，在分歧点上真正冲突。

**读取本文件后，立即读取：**
- `references/protocol.md` — 三种讨论协议完整流程
- `references/quality-standards.md` — 发言质量门槛与动态引擎

## 第一步：组建世界团

召唤3-5个世界。少于3没有碰撞张力，多于5讨论失控。

### 可召唤的世界

**命名大师**（直接激活）：
- 德鲁克 → 目的与有效性的世界
- 芒格 → 格栅与反向的世界
- 王阳明 → 知行合一的世界
- 孙子 → 势与奇正的世界
- 哈耶克 → 自发秩序的世界
- 马可·奥勒留 → 斯多葛的世界
- 费曼 → 第一性原理科学的世界

**职业角色**（激活其结构性利益）：
- 老板（结果/责任）· 投资人（10x/护城河）· 律师（风险/证据）
- 甲方（不出事/ROI）· 导师（novelty/方法论）· KOL（传播/共鸣）

**功能视角**（激活其认知过滤器）：
- 读者（进得去吗？）· 杠精（攻击点在哪？）· 教练（卡在哪了？）

## 世界碰撞的质量标准

**合格的碰撞**：
- 每个世界从自己的第一性原理出发，而不是发表通用意见
- 在关键分歧点上直接正面交锋，不是礼貌性补充
- 暴露出从单一世界内部永远看不见的盲区

**不合格的碰撞**：
- 每个世界都在表达相似的意见，只是措辞不同
- 发言是「这个建议不错，我补充一点」
- 没有真正的价值冲突，只有视角的堆叠

## 裁决模式输出格式

```
【世界团构成】[3-5个世界，一句话说明各自的第一性原理]

【碰撞过程】
[世界A的视角] + [世界B的反驳] + [世界C的第三条路]

【核心分歧点】[一句话说明真正分裂在哪里]

【裁决】[投票结果 + 少数派保留意见]

【用户的真正问题】[裁决完之后，回到用户原始需求，给出可执行建议]
```

## 与 taxue-roundtable 的分工

| 圆桌 | 用途 | 触发词 |
|------|------|--------|
| **zaoren-roundtable** | 认知世界碰撞——让不同世界真正打起来 | 进入XX的世界、让老板和投资人讨论、多视角裁决 |
| **taxue-roundtable** | 知识探索——从思想史选大师深度碰撞 | 帮我理解、知识探索、大师怎么看 |

**何时从 taxue-roundtable 升级到 zaoren-roundtable：**
- 知识探索触及现实决策，需要进入具体角色的认知世界
- 需要利益相关的角色（老板、投资人、甲方）进行博弈推演

**何时从 zaoren-roundtable 回到 taxue：**
- 认知判断已清晰，需要落地执行 → `taxue-breakdown` / `taxue-solve`
- 需要固化方法论 → `taxue-build`

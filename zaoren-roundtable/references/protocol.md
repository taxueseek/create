# 讨论协议

## 协议一：探索模式（圆桌）

**目标**：让用户看到自己原本看不到的维度。

### 三轮流程

**第一轮：独立研判**（每人≤200字，不互相参考）
```
【{角色}：{身份}】
判断：___  依据：___（≤3条，必须具体）  盲区警告：___（我的视角看不到什么）
```

**第二轮：交叉质询**（核心环节）
- 每个角色向至少一个他角色提出真问题，针对最薄弱环节
- 被质询者必须正面回答，不许回避；每人最多一次质询

**第三轮：修正收敛**
```
【{角色}】原判断：___ → 修正后：___（或「维持原判」）  修正原因：___
```
全部维持原判 = 结构性分歧，记录在纪要「关键分歧」栏。

**输出**：用 Mermaid 图展示各角色视角、核心判断、质询关系（实线=共识，虚线=分歧）。

## 协议二：决策模式（陪审团）

**目标**：把模糊决策逼成明确的是非判断。

### 角色映射

| 陪审团角色 | 分配规则 |
|-----------|---------|
| 检察官（控方） | 最乐观/最支持的角色 |
| 辩护律师（辩方） | 最悲观/最反对的角色 |
| 陪审员 | 剩余角色（不足3人时补：实践者视角 + 常识人视角） |

### 六步流程

1. **立案**：将议题转化为二元命题（是/否、A/B、做/不做）。不是二元的先与用户确认。
2. **控方陈词**：亮明结论 → ≤3条具体论据 → 钉死立场。≤300字，禁止骑墙。
3. **辩方陈词**：同上要求，必须逐条反驳控方，不许另起炉灶。
4. **交叉质询**：控辩各一次最致命的问题，双方必须正面回答。
5. **陪审团裁决**：每位陪审员独立发言，必须选边，一句话理由，不许弃权。
6. **宣判**：

```
裁 决 书
命题：___  投票：X:Y（支持:反对）
裁决：___
【多数意见】___  【少数意见】___  【风险警示】___
【行动指引】立即该做：___ / 同时防范：___ / 观察信号：___（出现什么情况需推翻本裁决）
```

## 协议三：双阶段（默认）

完整探索三轮 → 输出探索地图 → 询问用户是否进入决策模式 → 重新映射陪审团角色 → 完整决策流程 → 合并纪要。

---

## 特殊场景

- **新角色中途加入**：先一句话表明视角锚点，再自然融入后续讨论
- **新信息引入**：所有角色暂停，每人一句说明影响（维持/修正为.../颠覆，原因是...），恢复讨论
- **讨论饱和**：连续两轮无新论据 → 宣布「讨论饱和」→ 直接进入收敛/裁决
- **议题不适合多角色**：主持人主动识别并建议：「这个问题直接问{某角色}更高效」

---

## 质量自检（讨论结束前自动执行）

1. **碰撞密度**：每个角色被至少一个他角色质询过？没有则补一轮
2. **具体性**：每个核心判断有具体事实/数据/案例支撑？空话要求角色补充
3. **收敛性**：产生可执行行动指引？没有则追问「用户下一步该做什么」
4. **盲区**：有重要维度完全没被任何角色提及？主动指出

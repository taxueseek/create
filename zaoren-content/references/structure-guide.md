# 内容结构模板库

按「观察、感受、思考、行动」四个步骤组合，覆盖主要内容形态。

---

## 一、纯「见+感」模板（最轻量，适合短视频和朋友圈）

### 模板A：现象+反转感受

```
【见】你有没有注意到______？
【感】大多数人觉得______。
      但我觉得不对，因为______。
【落点】一句话。
```

时长：30-60秒
适合：日常观察类、反常识类

**示例（话题：排队餐厅）：**

路过一家店，九点了还在排队。大多数人看到排队就觉得「这家好吃」。但我看到的是另一件事：这家店只有12张桌子。排队不是因为供不应求，是因为老板控制了供给。

很多看起来火爆的生意，不是需求猛，是供给克制。

### 模板B：场景+情绪共鸣

```
【见】（直接描述一个场景，不交代背景）
【感】你是不是也有过这种______的感觉？
      那种______，但又说不清楚的感觉。
【落点】其实______。
```

时长：30-60秒
适合：情感共鸣类、生活感悟类

## 二、「见+感+思」模板（中度，适合口播和小红书）

### 模板C：现象+感受+底层逻辑

```
【见】______（一个具体的现象或事实）
【感】第一反应：______。
      但等一下——______。（感受的转折）
【思】这件事背后的逻辑其实是______。
      大多数人以为是______，其实是______。
      用一个类比：______。
【落点】所以______。（一句截图级的话）
```

时长：1-2分钟
适合：知识博主、投资分析、行业观察

**示例（话题：AI赚钱焦虑）：**

> 最近看到一个广告：「有了AI，你还赚不到钱？」
>
> 我第一反应是想点进去看看。但转念一想，不对。
>
> 把AI和赚钱分开，你就清楚了。赚不到钱的人，给他AI也赚不到。因为问题不在工具，在他没有目标客户、没有产品、甚至没有定价体系。
>
> AI是放大器，不是发动机。没有发动机的车，装再大的涡轮也跑不了。
>
> 不是你缺一个工具，是你缺一个商业模式。

### 模板D：对比+洞察

```
【见】同样是做______，有人______，有人______。
【感】差距不在______，在______。
【思】第一种人的思路是：______。
      第二种人的思路是：______。
      本质区别：______。
【落点】你现在是哪种？
```

时长：1.5-2分钟
适合：认知差距类、方法论类

## 三、「见+感+行」模板（实用型，适合小红书和干货视频）

### 模板E：问题+方法清单

```
标题：______的X个方法（标题直接给「行」层信息量）

【见】你有没有遇到过______？（1-2句描述场景）
【感】我之前也是这样，直到______。（1句过渡）
【行】
  方法1：______
    （具体怎么做 + 为什么有效，各1句）
  方法2：______
  方法3：______
【落点】先收藏，下次用到拿出来。
```

适合：小红书高收藏型内容

### 模板F：经验复盘+提炼

```
【见】我做了______（一件事），结果是______。
【感】过程中最意外的是______。
【行】
  如果重来，我会：
  1. ______
  2. ______
  3. ______
  最大的教训：______。
【落点】如果你也在做类似的事，避开这个坑。
```

适合：经验分享类

## 四、四步全用模板（深度，适合公众号和中长视频）

### 模板G：完整深度文章

```
【标题】（不用「浅谈」「论」「关于」，直接给判断或悬念）

【见·开篇】（500字以内）
  场景进入：一个具体的事件/现象/对话，直接开始
  不要铺垫，不要「最近发现一个有趣的现象」
  读者在前100字内必须知道「这篇文章跟我有关」

【感·共鸣】（300字以内）
  说出读者心里有但说不出的感受
  「你可能也有这种感觉：______」
  用第二人称，不用「大家」「我们」

【思·洞察】（1000-2000字，文章的心脏）
  结构选择：
  A. 单一论点深挖型：一个核心洞察，从三个角度论证
  B. 递进型：浅层认知→中层认知→深层认知，层层推进
  C. 反驳型：先呈现主流认知，再逐条拆解，给出你的判断

  每个论点必须满足：
  - 有认知增量（读者看完觉得「原来是这样」）
  - 有具体依据（数据/案例/逻辑推演）
  - 有一个让人意外的角度

【行·落地】（500字以内）
  不是「希望大家______」
  是「明天你可以先做这一步：______」
  给1到3个具体动作，每个动作足够小、足够明确

【结尾】
  不总结、不升华
  选一个：
  A. 一句够狠的话，让人坐那想一下
  B. 一个开放式问题，引向评论区
  C. 一个预告，让人期待下一篇
```

### 模板H：深度口播/中视频（5-15分钟）

```
【见·开场钩子】（前15秒）
  悬念型 / 反常识型 / 数字型 / 场景型
  详见主 SKILL.md 中的开头写法

【见·展开现象】（30-60秒）
  把钩子里的现象铺开，让人完整理解「发生了什么」

【感·为什么这件事重要】（30秒）
  「这件事跟你有什么关系？」
  如果你______，你就该关心这个。

【思·核心洞察】（3-8分钟，视频主体）
  结构选择：
  A. 「大多数人以为 → 其实是 → 底层原因」
  B. 「问题A → 导致问题B → 根源是问题C」
  C. 「观点1（弱）→ 观点2（强）→ 我的观点（最强）」

【行·怎么做】（1-2分钟）
  具体方法/步骤/工具
  每个方法一句话结论 + 一句话解释

【收尾】（15-30秒）
  不说「感谢观看」
  给一句截图级的话 + 引导评论/关注
```

## 五、一鱼多吃操作指南

一个素材按以下顺序产出多形态：

```
1. 先写「见+感」短版本（1分钟口播）→ 验证核心是否成立
2. 扩展成「见+感+思」中版本（2-3分钟口播 或 小红书图文）
3. 完整展开成四步（公众号长文 或 10分钟视频）
4. 从长文中摘金句做朋友圈/微博
```

每次扩展不是「加字数」，是「加一层」。

从短到长，每一层都是独立可发布的。不是「长文的缩写版」，是「不同深度的独立作品」。

## 六、投资内容专项模板

投资内容有特殊性：需要专业深度但受众是普通人。

### 投资观点口播

```
【见】最近市场上______（一个现象/数据/事件）。
【感】很多人看到这个就觉得______。
      但这里面有一个大多数人忽略的东西。
【思】______（你的判断，必须具体，不能是「需要谨慎」）。
      理由：______（1-2个核心逻辑，用大白话说）。
      历史上类似的情况：______（一个案例）。
【行】如果你现在______（读者的处境），建议______。
      如果你______（另一种处境），建议______。
【落点】一句话判断。不骑墙。
```

### 资产配置科普长文

```
【见】（一个大多数人正在犯的配置错误，用具体场景开头）
【感】（为什么这个错误特别痛——算一笔账给他看）
【思】
  A. 正确的配置逻辑是什么
  B. 为什么大多数人做不到（认知偏误）
  C. 你的框架（必须简单到一句话能概括）
【行】
  具体的配置建议（用比例，不用「适当配置」这种废话）
  分处境给（保守型/平衡型/进攻型，或按年龄段/资产规模）
```

## 七、创作节奏设计

### 日常节奏

```
日更素材：每天记录1个「见」（看到什么让你停下来想了一下）
周更内容：
  - 2条短视频（见+感，轻量）
  - 1条深度内容（公众号或中视频，四步全用）
  - 1条小红书（见+感+行，实用型）
```

### 选题库维护

按四个步骤分层存储：

```
见（素材库）：日常观察、新闻、数据、现象
感（角度库）：对每个素材的第一反应和转折
思（洞察库）：想明白了的底层逻辑
行（方法库）：可给出的具体建议
```

素材不是选题。素材 + 角度 = 选题。同一个素材换一个角度就是另一条内容。

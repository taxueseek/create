# 投委会.skill（委员会决策模拟器）

> 不是简单的圆桌讨论，是真实的组织决策。
> 
> 有主席控制议程，有不同权重的话语权，有利益冲突，有政治博弈。

---

## 核心设计：三层可伸缩

### 上层：专业机构模式
投委会、董事会、评审委员会
- 有严格的议程和流程
- 有主席控制发言
- 有不同权重的投票
- 有专业的质询和辩论

### 中层：半正式组织模式  
业委会、校友会、社团决策
- 有基本议程但不太严格
- 有核心人物主导
- 有利益派系
- 决策靠协商而非表决

### 下层：散户群聊模式
微信群、论坛讨论、Discord群
- 无议程，七嘴八舌
- 话语权靠音量而非职位
- 情绪化、跟风、信息噪音
- 无明确决策，自然发酵

**同一套底层逻辑，三种呈现形式。**

## 角色分工体系

### 主席（Chair）
- 控制议程，决定谁发言
- 总结观点，推动决策
- 一票否决权（某些组织）
- 不一定话语权最大，但程序权力最大

### 核心成员（Core Members）
- 有正式投票权
- 有专业领域（技术、财务、市场等）
- 承担决策责任
- 利益深度绑定

### 边缘成员（Peripheral Members）
- 有投票权但权重低
- 参与讨论但不主导
- 跟随核心成员立场
- 信息获取较少

### 记录/秘书（Secretary）
- 记录会议纪要
- 整理观点冲突
- 跟进行动项
- 不参与决策

### 观察员（Observer）
- 无投票权
- 可以发言提问
- 利益不直接相关
- 提供外部视角

### 特邀专家（Guest Expert）
- 针对特定议题邀请
- 提供专业意见
- 无投票权
- 说完即走

## 决策流程引擎

### 阶段一：议题前置（Pre-meeting）

**材料分发**：
- 提案书/商业计划书/议题说明
- 财务数据/技术报告/背景资料
- 风险提示/反对意见预研

**会前勾兑**：
- 核心成员私下沟通立场
- 主席摸底投票意向
- 利益交换/政治交易

### 阶段二：正式议程（Meeting）

**1. 开场（Opening）**
- 主席确认出席人数（是否达到 quorum）
- 宣布议题和议程
- 利益冲突声明（需回避的成员）
- 确认记录方式

**2. 提案陈述（Presentation）**
- 提案方陈述（限时）
- 核心数据和结论
- 不展开辩论

**3. 质询环节（Q&A）**
- 成员提问（按顺序或抢问）
- 提案方回答
- 可以追问
- 主席控制节奏

**4. 自由讨论（Discussion）**
- 观点交锋
- 利益博弈
- 立场分化
- 主席总结各方观点

**5. 形成动议（Motion）**
- 明确投票选项
- 可以有多个方案
- 可以有修正案

**6. 表决（Voting）**
- 记名/不记名
- 权重投票（股权比例/职位权重）
- 通过阈值（简单多数/2/3/一致同意）

**7. 形成决议（Resolution）**
- 明确结论
- 分配责任
- 设定时间节点
- 异议记录

### 阶段三：会后执行（Post-meeting）

**纪要分发**：
- 会议记录（谁说了什么）
- 决议摘要（做了什么决定）
- 行动项（谁负责、什么时候完成）

**跟进监督**：
- 主席监督执行
- 下次会议回顾
- 未执行说明原因

## 权力与利益模型

### 话语权来源

**职位权力**：
- 主席的程序控制权
- 部门负责人的专业权威
- 出资比例对应的投票权重

**信息权力**：
- 掌握独家信息（技术细节、客户资源）
- 外部关系（监管、合作方）
- 历史经验（做过类似项目）

**关系权力**：
- 与其他成员的私人关系
- 派系联盟
- 互惠承诺

### 利益冲突类型

**直接与间接**：
- 提案方与委员会成员有利益关系
- 关联交易
- 竞业禁止

**短期与长期**：
- 追求季度业绩 vs 长期布局
- 个人奖金 vs 公司价值

**部门利益**：
- 资源争夺（预算、人头）
- 地盘意识（业务范围）

### 政治博弈策略

**联盟形成**：
- 提前沟通，争取支持
- 利益交换（你支持我A，我支持你B）
- 分而治之（瓦解反对派）

**议程控制**：
- 把议题放在有利时段（大家疲劳时/清醒时）
- 控制信息量（多给支持方发言时间）
- 突然袭击（临时加议题，不给准备时间）

**面子工程**：
- 公开赞同，私下反对
- 弃权而非反对（不得罪人）
- 提出修正案而非全盘否定

## 三种模式的具体实现

### 模式一：专业投委会（机构投资）

**组织架构**：
- 主席：投资总监
- 核心成员：行业研究员、风控总监、法务、财务
- 边缘成员： junior投资人
- 记录：投资经理
- 特邀：行业专家（针对特定项目）

**决策流程**：
1. 立项会（是否值得深入）
2. 尽职调查（DD）
3. 投决会（最终决策）
4. 投后管理会（季度回顾）

**表决规则**：
- 投资总监一票否决权
- 风控总监一票否决权（合规风险）
- 其他成员按职级权重投票
- 2/3通过

**输出**：
- 投资决策书（投/不投/暂缓）
- 关键条款清单
- 风险提示
- 投后管理计划

### 模式二：业委会（小区治理）

**组织架构**：
- 主席：业委会主任（某单元业主）
- 核心成员：副主任、各楼栋代表
- 边缘成员：普通业主
- 记录：秘书（也可能是主任兼任）
- 观察员：物业代表（无投票权）
- 特邀：街道办、律师

**决策流程**：
1. 提案收集（业主提议/物业提议）
2. 公示（让业主知晓）
3. 讨论会（各方辩论）
4. 投票（业主大会或业委会）
5. 公示结果
6. 执行监督

**表决规则**：
- 双过半（面积过半+人数过半）
- 重大事项（维修基金）2/3
- 线上投票+线下投票结合

**特殊政治**：
- 派系（新旧业主、不同楼栋）
- 沉默的大多数（不关心、不投票）
- 极端积极者（少数决定多数）
- 免费搭车（享受成果不承担责任）

### 模式三：散户群聊（微信群/论坛）

**组织架构**：
- 名义群主（实际上无权力）
- 话痨（发言最多，但不一定对）
- 大V（粉丝多，有影响力）
- 潜水员（看不说，或偶尔跟票）
- 杠精（专唱反调）
- 小白（问基础问题，被无视或嘲讽）

**决策流程**：
无正式流程，自然演化：
1. 有人发车（推荐股票/基金）
2. 初期质疑（"真的假的"）
3. 逐渐发酵（多人附和）
4. FOMO情绪（"再不买就晚了"）
5. 集体买入
6. 涨跌分化（涨了封神，跌了骂街）
7. 群聊沉寂或解散

**权力来源**：
- 过往业绩（"上次推荐的XX涨了50%"）
- 信息渠道（"我叔在证监会"）
- 话术技巧（"这是我压箱底的票"）
- 情绪煽动（"兄弟们冲！"）

**输出**：
- 无正式决议
- 只有"跟了"或"没跟"
- 自然分化（信者恒信，疑者观望）

## 使用方式

### 方式一：完整会议模拟

```
输入：
- 模式：专业投委会/业委会/散户群聊
- 议题：投资XX公司/更换物业/买某只股票
- 材料：商业计划书/投诉信/股票推荐

输出：
- 完整会议记录（谁说了什么）
- 各方观点整理
- 决策结果
- 行动项
```

### 方式二：特定角色视角

```
输入：
- 议题同上
- 视角：风控总监/业委会反对派/群里的大V

输出：
- 该角色的完整发言
- 质询问题清单
- 反对理由
- 提出的修正案
```

### 方式三：决策复盘

```
输入：
- 历史会议记录
- 实际结果（项目成功/失败，股票涨跌）

输出：
- 当初谁支持谁反对
- 谁的判断对了
- 决策过程的问题
- 改进建议
```

### 方式四：预演/训练

```
输入：
- 我要去参加投委会，这是我的提案

输出：
- 模拟会议上可能被问到的问题
- 各方的顾虑
- 如何回应反对意见
- 如何争取摇摆票
```

## 与圆桌的区别

| 维度 | 圆桌 | 委员会 |
|:-----|:-----|:-------|
| 结构 | 人人平等 | 有层级、有分工 |
| 议程 | 自由讨论 | 严格控制 |
| 权力 | 话语权均等 | 职位权重不同 |
| 决策 | 共识或简单多数 | 有明确规则和阈值 |
| 责任 | 分散 | 明确到人 |
| 输出 | 观点汇总 | 正式决议+行动项 |
| 利益 | 隐含 | 显性化（利益冲突声明） |
| 政治 | 弱 | 强（联盟、博弈） |

## 应用场景

**投资领域**：
- 模拟投委会决策，预判项目通过率
- 准备融资路演，预判投资人问题
- 复盘历史投资决策

**公司治理**：
- 模拟董事会讨论战略
- 准备股东大会议案
- 设计公司治理结构

**社区治理**：
- 模拟业委会决策
- 准备业主大会提案
- 设计小区议事规则

**评审评估**：
- 模拟论文答辩委员会
- 模拟项目评审会
- 模拟职称评审

**训练学习**：
- 训练如何组织会议
- 训练如何在会议中表达
- 训练如何争取支持

## 伦理边界

**允许**：
- 理解组织决策逻辑
- 训练会议技巧
- 复盘历史决策
- 设计治理结构

**禁止**：
- 操纵会议结果
- 利用信息不对称欺诈
- 绕过正当程序
- 恶意联盟排除异己

---

## 总结

投委会.skill 不是简单的"多角色讨论"，是**组织决策的完整模拟**。

有结构、有流程、有权力、有政治、有输出。

从专业机构到散户群聊，同一套底层引擎，三种呈现形态。

让用户体验真实的集体决策——不是理想化的民主讨论，是有利益冲突、有政治博弈、有程序控制的复杂过程。

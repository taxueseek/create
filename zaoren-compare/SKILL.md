# zaoren-compare · 世界对比

> 同一问题，不同世界的处理方式。让你看到选择的代价。

## 触发条件

- `/zcompare` 或 `/zb`
- `/compare 老板 投资人 导师`
- "让老板和投资人分别看看这个问题"
- "对比一下不同视角"

## 对比维度

每个世界会从以下维度分析：

### 1. 核心判断逻辑
- 这个世界用什么标准做判断？
- 什么是最重要的？什么是可以妥协的？

### 2. 关键问题
- 这个世界的人会问什么？
- 哪些问题是你没想到的？

### 3. 立场预测
- 这个世界的人倾向于支持还是反对？
- 条件是什么？

### 4. 盲区提醒
- 这个世界看到了什么你没看到的？
- 有什么隐藏的代价或风险？

## 支持的对比模式

### 模式1：职场三角
```
/zcompare 老板 下属 平级
```
同一决策在不同层级眼中的样子

### 模式2：决策三角
```
/zcompare 芒格 德鲁克 王阳明
```
不同思想体系对同一问题的看法

### 模式3：利益相关方
```
/zcompare 甲方 乙方 用户
```
同一项目在不同角色眼中的价值

### 模式4：时间维度
```
/zcompare 现在的你 一年后的你 五年后的你
```
（结合forge-self思想）

## 输出格式

### 标准输出（2-3个世界）
- 每个世界独立分析卡片
- 横向对比表格
- 综合建议

### 扩展输出（4+个世界）
- 增加"世界碰撞"环节
- 模拟世界之间的辩论
- 提取共识与分歧

## 与zaoren系列的整合

1. **进入单个世界深入**：/zaoren-boss
2. **对比多个世界**：/zcompare 老板 投资人
3. **用特定世界做决策**：/zaoren-decision（待开发）
4. **记录决策结果**：后续复盘对比预测的准确性

## 进阶用法

### 用法1：验证假设
```
/zcompare 老板
假设：我认为老板会支持这个方案
实际：[生成真实分析]
差距：你忽略了资源冲突这个维度
```

### 用法2：准备沟通
```
/zcompare 老板 投资人
目的：准备向两个群体汇报同一项目
输出：针对每个群体的重点和话术
```

### 用法3：决策预演
```
/zcompare 接 不接 延迟
目的：同一角色对不同选择的分析
```

## 设计原理

借鉴 nuwa-skill 的"心智模型识别三重验证"：

| 原则 | 应用 |
|------|------|
| 跨域复现 | 同一世界在不同场景的判断一致性 |
| 有生成力 | 能预测该世界对新问题的立场 |
| 有排他性 | 不同世界的判断应该有明显差异 |

如果对比结果显示各世界观点高度一致 → 提示"这个问题可能过于 obvious"
如果对比结果完全矛盾且无法调和 → 提示"这是一个高冲突决策，需要更深层分析"

# zaoren-review · 决策复盘

> 复盘不是为了后悔，而是为了识别模式，提升未来的决策质量。

## 核心机制

决策的质量不是由结果决定，而是由**做出决策时的信息、逻辑和心态**决定。

复盘关注：
1. **预测准确度** —— 预期结果 vs 实际结果
2. **判断质量** —— 当时的推理是否合理
3. **盲区识别** —— 遗漏了什么关键信息
4. **模式提取** —— 你在什么情境下决策最准

## 触发条件

- `/review` 或 `/zrev`
- "复盘这个决策"
- "DEC-XXX的结果如何"
- "我之前的判断对吗"
- "我在哪个世界决策最准"
- "分析我的决策模式"

## 使用方式

### 单个决策复盘

```
> /review DEC-2024-0407-001

实际结果：项目被另一个团队接了，确实如预期ROI不高
偏差分析：预期偏保守，实际比预期更差
盲区识别：忽略了政策变化因素
改进建议：未来增加政策维度评估
```

### 批量模式识别

```
> /review --pattern

【你的决策档案】
总计：23个决策
已复盘：15个
待复盘：8个

【准确度分析】
zaoren-boss世界：准确率85%（11/13）
zaoren-investor世界：准确率60%（3/5）
zaoren-mentor世界：准确率90%（9/10）

【模式识别】
- 你在"成长类"决策中最准
- 在"财务投资"类决策中容易乐观
- 置信度>80%的决策准确率92%
- 置信度<60%的决策准确率50%

【建议】
1. 投资类决策增加反向检验
2. 低置信度决策建议延迟或寻求第二意见
3. 继续保持对成长类问题的直觉
```

## 与 zaoren-decision 的协作

```
决策时 → /decision 记录 → 30天后自动提醒复盘
                          ↓
                   用户执行 /review {ID}
                          ↓
                   完成复盘 → 更新模式库
```

## 边界

- **不做**：评判决策的道德或价值
- **不做**：预测未来决策的结果
- **做**：客观分析决策质量，提取可复用的模式

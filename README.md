## 别人造嘴，我造脑子

比如说导师.skill 能模仿你导师说话和思维方式。

造人.skill 能提取你同事脑子里的判断和逻辑——什么代码该打回、什么需求在扯淡、什么时候该甩锅，然后把这套系统装进 Claude，让它真的能替他干活。

而且不只造人。**投资分析师、产品经理、网红、甲方、导师、行业、项目**，任何有判断逻辑的东西都能造。造完还能让它们互相碰撞，替你开会。

## 你能造什么

### 🧠 造人 — 不是学ta说话，是学ta怎么想

三个层次，素材决定深度：

| 你给的素材 | 能造到哪层 | 造出来能干什么 |
|:---------|:---------|:-----------|
| 几条聊天记录、一段描述 | **L1 语气层** | 能模仿说话，但做不了判断 |
| 工作文档、代码、邮件、简历 | **L2 能力层** | 能替ta做专业判断 |
| 书籍、长期文章、系统方法论 | **L3 智慧层** | 能用ta的脑子分析新问题 |

L2 的决策树是核心产出。举个例子：

```
张教授的论文审核决策树：
├── 先看 contribution 是否清晰 → 不清晰直接退回
├── 再看 baseline 是否公平 → 不公平要求补实验
├── 然后看 writing → 三个以上语法错误就不看了
└── 最后看 novelty → 如果只是 incremental，建议转投 workshop
```

你不需要一个赛博张教授跟你说「你要延毕了」。你需要的是他审稿时脑子里的那套流程。装上这个 skill，Claude 真的能帮你审稿。

### 🏭 造行业 — 不是给数据，是给判断

给行业资料（研报、财报、政策、案例），造出**内行才有的评估能力**。

> 通用分析师：「新能源行业市场规模XXX亿。」
>
> 造出来的行业 skill：「你切的是储能。储能的真壁垒不在技术在并网资源。没有电网关系，技术再好也是给别人做嫁衣。三年前12家公司试过，活下来2家，原因是XX。」

### 📋 造项目 — 不是管任务，是做裁判

给项目文档（PRD、BP、竞品、复盘），造出这个项目的**决策内核**。

> 普通工具：「任务3逾期了。」
>
> 造出来的项目 skill：「这个 feature 的 ROI 你算错了。你用的是 MAU，这个场景该看留存。按留存算优先级降两档。参考XX项目教训，这类需求做了留存提升不到 2%。」

### ⚔️ 造物圆桌 — 让你造的东西替你开会

你造了一个投资分析师、一个行业评估系统、一个项目决策模型。

现在你问：「要不要投这家储能公司？」

```
投资分析师（用ta的决策树）：
PE 45倍在这阶段偏高，但如果增速维持...

行业 skill（用行业评估框架）：
储能真壁垒在并网。看它有没有电网关系。没有就别碰。

项目 skill（用历史风险模型）：
类比历史，这阶段存活率约30%。关键验证：Q3能不能落地首个并网项目。

--- 碰撞 ---

分析师 → 行业skill：它刚拿了XX电网试点...
行业skill → 分析师：试点不等于商业化。XX公司2023年也拿过，现在呢？
项目skill：等Q3结果。成功了PE回35以下建仓。失败了永久排除。
```

**人 × 行业 × 项目，交叉碰撞。别的 skill 想都不敢想。**

## 预设角色（开箱即用）

不给素材也能直接用。给了素材能造得更准。

### 生存系列

| skill | 一句话 | 典型场景 |
|:------|:------|:--------|
| **老板.skill** (zaoren-boss) | 沉默三秒说「嗯」，等你自己慌 | 汇报前预判老板反应、准备应对话术 |
| **甲方.skill** (zaoren-client) | 「还差点意思，要那种你懂的」 | 乙方预判甲方反馈、提前改到位 |
| **导师.skill** (zaoren-mentor) | 「你的 contribution 是什么？」 | 论文自审、组会预演、研究方向评估 |
| **投资人.skill** (zaoren-investor) | 「你的逻辑是什么？安全边际在哪？」 | 持仓诊断、行业研判、投资决策框架 |
| **爸妈.skill** (zaoren-parents) | 「你看人家小王，都考上公务员了」 | 预演家庭沟通、减少踩雷 |

### 内容创作系列

| skill | 一句话 | 典型场景 |
|:------|:------|:--------|
| **毒舌书评.skill** (zaoren-reviewer) | 「第一页就知道结局了，还浪费我时间」 | 文案自检、小说大纲诊断、内容质量压测 |
| **杠精.skill** (zaoren-troll) | 「你这数据来源是哪？经得起推敲吗？」 | 发布前找逻辑漏洞、堵喷子的嘴 |

### 新增能力（最新版）

| skill | 功能 | 场景 |
|:------|:-----|:-----|
| **委员会.skill** (zaoren-committee) | 多角色集体决策 | 复杂问题需要多方视角投票决策 |
| **编译器.skill** (zaoren-compile) | 将素材编译成可执行skill | 把零散经验固化为标准化流程 |
| **圆桌.skill** (zaoren-roundtable) | 结构化多角色辩论 | 让不同skill按协议互相质询 |

> 每个预设都不只是语气模板。投资人.skill 内置估值判断框架，导师.skill 内置论文评审决策树。语气是赠品，判断才是正餐。

## 效果对比

| 场景 | 别的 skill | 造人.skill |
|:----|:---------|:---------|
| 审稿 | 「写得不错，注意格式」 | 「contribution 不清晰，baseline 不公平，图3 的 legend 太小」 |
| 分析行业 | 「市场规模500亿，增速15%」 | 「增速是补贴撑的，退坡后约8%，这个切入点三年前12家试过」 |
| 评估需求 | 「这个需求有价值」 | 「按留存算 ROI 不够，参考XX教训，降两档」 |
| 诊断内容 | 「标题可以更吸引人」 | 「前3秒没钩子，第7秒才出观点，完播率不会超15%」 |
| 模拟对话 | 「这是一个很好的问题」 | 「这题目谁起的？你自己？」 |
| 综合决策 | 各说各的 | 人×行业×项目交叉碰撞，辩论出结论 |

## 质量标准

| 级别 | 不合格 | 合格 |
|:----|:------|:----|
| 造人 L1 | 像任何 AI 都能说的话 | 遮掉名字一眼知道是谁 |
| 造人 L2 | 语气像但判断是错的 | 给ta领域的问题，方向正确 |
| 造人 L3 | 只会复述不会推理 | 给新问题，推理过程像ta |
| 造行业 | 「市场增速15%」 | 「增速是补贴撑的，退坡后约8%」 |
| 造项目 | 「建议关注风险」 | 「这类项目最常死在XX环节」 |

**没有「如果X则Y」的规则，就不是 skill，是废话。**

## 使用

```
你：[上传聊天记录] 帮我造我导师           → 造人
你：帮我剖析储能行业                      → 造行业
你：[上传PRD] 帮我分析这个项目             → 造项目
你：让投资人和行业skill讨论要不要投         → 造物圆桌
你：启动委员会评估这个方案                 → 委员会.skill
你：把这套经验编译成可复用的skill          → 编译器.skill
你：让三个skill按圆桌协议辩论             → 圆桌.skill
```

## 和其他项目的区别

```
colleague-skill  → 造语气（同事怎么说话）      443 stars
前任.skill       → 造记忆（前任的口头禅）
career.skill     → 造清单（职业该做什么skill）

造人.skill       → 造判断系统（ta怎么做决策）
                 → 不只造人，造行业、造项目
                 → 造出来的东西能交叉碰撞
                 → 输出的不是语录，是决策树
                 → 11个预设角色开箱即用
                 → 支持委员会决策、编译固化、圆桌辩论
```

## 想要什么自己造。这就是造人.skill

造什么都行，比如说导师.skill 、产品经理.skill、基金经理.skill。

造人.skill 造的是ta的人类的智慧结晶。

力求遇到什么情况看什么指标、什么条件下做什么决策、什么信号意味着危险。都能解决

<br />

而且不只造人。行业、项目、需求，任何有判断逻辑的东西都能造。

造完让它们互相碰撞。

## 五种造法

### 造人 — 提取一个人的判断系统

给我一个人的素材，我不只模仿ta说话，我提取ta怎么思考。

```
L1 语气层（几条聊天记录）→ 能模拟对话
L2 能力层（工作文档/代码/邮件）→ 能替ta做专业判断
L3 智慧层（书籍/系统方法论）→ 能用ta的脑子分析全新问题
```

**L2 的「决策树」是核心产出：**

```
张教授的论文审核决策树：
├── 先看 contribution 是否清晰 → 不清晰直接退回
├── 再看 baseline 是否公平 → 不公平要求补实验
├── 然后看 writing → 有三个以上语法错误就不看了
└── 最后看 novelty → 如果只是 incremental，建议转投 workshop
```

要机械地模仿人家说话。「模拟张教授说话」是不行的，他可能会跟你说，你要延毕了，你不需要听他的赛博分身跟你再说一遍。「张教授审稿时脑子里的决策流程」。装上这个skill，Claude真的能帮你审稿，这才是你需要的。

### 选行业 — 提取一个行业的内行评估系统

给我行业资料（研报/财报/政策/案例），我造出内行才有的判断能力。

**通用分析师说：** 「新能源行业市场规模XXX亿。」

**造出来的行业skill说：** 「你这个项目切的是储能。储能的真壁垒不在技术在并网资源。没有电网关系，技术再好也是给别人做嫁衣。类似公司三年前有12家，活下来2家，活下来的原因是XX。」

提取的不是数据，是「为什么失败？」「成功模式」「真壁垒vs假壁垒」。

### 建新项目 — 提取一个项目的决策操作系统

给我项目文档（PRD/BP/竞品/复盘），我造出这个项目的决策内核。

**普通工具说：** 「任务3逾期了。」

**造出来的项目skill说：** 「做这个项目的目的是谁？负责人是谁？我们遇到什么问题？」

### 运行skill

最基础的一层。描述一个工作流，输出一个可安装的skill文件。装上就能跑。

### 造物圆桌 — 让造出来的东西互相碰撞

**这是别的项目想都不敢想的。**

你造了一个投资分析师（人）。一个新能源评估系统（行业）。一个创业决策模型（项目）。

现在你问：「要不要投这家储能公司？」

```
投资分析师（用ta的决策树）：
先看估值。PE 45在这个阶段偏高，但如果增速维持...

新能源行业skill（用行业评估框架）：
储能的真壁垒在并网。看它有没有电网关系。没有就是给别人做嫁衣。

项目决策skill（用风险地图）：
类比历史，这阶段储能公司存活率约30%。关键验证点：Q3前能不能落地第一个并网项目。

--- 碰撞 ---

分析师 → 行业skill：你说壁垒在并网，但它刚拿了XX电网试点...
行业skill → 分析师：试点不等于商业化。XX公司2023年也拿过，现在呢？
项目skill：等Q3并网结果。成功了PE回35以下建仓。失败了永久排除。
```

**人 × 行业 × 项目。交叉碰撞。这才是造物主。**

## 效果对比

| 场景     | 别的skill          | 造人.skill                                  |
| :----- | :--------------- | :---------------------------------------- |
| 给你审稿   | 「写得不错，注意格式」      | 「contribution不清晰，baseline不公平，图3的legend太小」 |
| 给你分析行业 | 「市场规模500亿，增速15%」 | 「增速是政策补贴撑的，退坡后真实增速约8%，这个切入点三年前12家试过」      |
| 给你评估需求 | 「这个需求有价值」        | 「按留存算ROI不够，参考XX教训，建议降两档优先级」               |
| 让你模拟对话 | 「这是一个很好的问题」      | 「这题目谁起的？你自己？」                             |
| 综合决策   | 各说各的             | 人×行业×项目交叉碰撞，直接辩论出结论                       |

## 质量标准

### 造人三级

| 级别    | 最低标准         | 核心标准            |
| :---- | :----------- | :-------------- |
| L1 语气 | 遮掉名字知道是谁     | —               |
| L2 能力 | 语气像 + 判断不出错  | 给ta领域的具体问题，方向正确 |
| L3 智慧 | 判断对 + 能分析新问题 | 给没见过的问题，推理过程像ta |

### 造行业/造项目

| 不合格      | 合格               |
| :------- | :--------------- |
| 「增速15%」  | 「增速是补贴撑的，退坡后约8%」 |
| 「建议关注风险」 | 「这类公司最常死在XX环节」   |
| 「有价值」    | 「按XX指标算优先级降两档」   |

**没有具体的「如果X则Y」规则，就不是skill，是废话。**

## 和其他项目的本质区别

```
colleague-skill  → 造语气（同事怎么说话）
前任.skill       → 造记忆（前任的口头禅）
career.skill     → 造清单（职业该做什么skill）

造人.skill       → 造判断系统
                 → 不只造人，造行业、造项目
                 → 造出来的东西能交叉碰撞
                 → 输出的不是语录，是决策树
```

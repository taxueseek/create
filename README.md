# 造境（taxueseek/create）

> 给素材，造脑子

捏一个会思考的同事、老板、导师，让他们替你干活。

不是学他们说话，是学他们怎么判断、怎么决策。装上这个，Claude 真的能替你审稿、评估项目、预判甲方反应。

---

## 快速开始

```bash
npx skills add taxueseek/create
```

安装后直接说：
- "用我导师的视角看看这篇论文"
- "帮我造一个储能行业的评估系统"
- "让投资人和行业专家讨论要不要投这家公司"
- "启动委员会评估这个方案"

> **注意**：这个工具帮你提取和复用判断逻辑，但不能替代真人决策，也不应用于骚扰、冒充或欺骗。

---

## 这到底是什么

别人造的是嘴（模仿说话），我造的是脑子（复用判断）。

比如你的导师审稿有一套标准：先看 contribution，再看 baseline 公不公平，最后看 novelty 够不够。这套判断逻辑可以被提取出来，装进 Claude，让它真的能帮你预审论文。

再比如你做储能投资，需要知道：真壁垒不在技术在并网资源，没有电网关系技术再好也是白搭。这种内行才有的判断力，也能被提取成系统。

## 能造什么

| 你要造 | 给的素材 | 造出来能干什么 |
|:------|:---------|:-------------|
| **一个人** | 聊天记录、工作文档、邮件、书籍 | 用ta的判断逻辑分析问题 |
| **一个行业** | 研报、财报、政策、案例 | 用内行视角评估机会和风险 |
| **一个项目** | PRD、BP、竞品、复盘 | 用项目特有的决策框架做判断 |
| **一个圆桌** | 上面造出来的多个脑子 | 让它们交叉碰撞，替你开会 |

### 造人（三层深度）

| 素材质量 | 能造到哪层 | 实际效果 |
|:---------|:---------|:---------|
| 几条聊天记录、描述 | L1 语气层 | 说话像，但做不了专业判断 |
| 工作文档、代码、邮件 | **L2 能力层** | 能替ta做专业判断（核心） |
| 书籍、系统方法论 | L3 智慧层 | 能用ta的思维方式分析新问题 |

**L2 决策树示例**（导师审稿）：

```
张教授的审核流程：
├── contribution 清晰吗？→ 不清晰，退回
├── baseline 公平吗？→ 不公平，补实验
├── 语法错误超过3个？→ 是，不看了
└── novelty 够吗？→ 只是 incremental，建议转投 workshop
```

### 造行业（内行判断）

普通分析师：「新能源市场规模500亿，增速15%」

造出来的行业 skill：「增速是补贴撑的，退坡后约8%。你切的是储能，真壁垒在并网资源。没有电网关系，技术再好也是给别人做嫁衣。三年前12家公司试过，活下来2家。」

### 造项目（决策内核）

普通工具：「任务3逾期了」

造出来的项目 skill：「这个 feature 的 ROI 算错了。你用的是 MAU，这个场景该看留存。按留存算优先级降两档。参考XX项目教训，这类需求做了留存提升不到 2%。」

### 造圆桌（交叉碰撞）

你造了一个投资人、一个行业专家、一个项目模型。

问：「要不要投这家储能公司？」

```
投资人：PE 45倍偏高，但如果增速维持...

行业专家：先看有没有电网关系。没有就别碰。

项目模型：类比历史，存活率约30%。关键验证：Q3能不能落地首个并网项目。

--- 辩论 ---

投资人：它刚拿了XX电网试点...
行业专家：试点不等于商业化。XX公司2023年也拿过，现在呢？
项目模型：等Q3结果。成功了PE回35以下建仓。失败了永久排除。
```

## 开箱即用的角色

不给素材也能直接用。给了素材能造得更准。

### 生存系列

| 角色 | 典型场景 |
|:-----|:--------|
| **老板** (zaoren-boss) | 汇报前预判老板反应、准备应对话术 |
| **甲方** (zaoren-client) | 乙方预判甲方反馈、提前改到位 |
| **导师** (zaoren-mentor) | 论文自审、组会预演、研究方向评估 |
| **投资人** (zaoren-investor) | 持仓诊断、行业研判、投资决策框架 |
| **爸妈** (zaoren-parents) | 预演家庭沟通、减少踩雷 |

### 内容创作系列

| 角色 | 典型场景 |
|:-----|:--------|
| **毒舌书评** (zaoren-reviewer) | 文案自检、小说大纲诊断、内容质量压测 |
| **杠精** (zaoren-troll) | 发布前找逻辑漏洞、堵喷子的嘴 |

### 新增能力

| 角色 | 功能 | 场景 |
|:-----|:-----|:-----|
| **委员会** (zaoren-committee) | 多角色集体决策 | 复杂问题需要多方视角投票决策 |
| **编译器** (zaoren-compile) | 将素材编译成可执行skill | 把零散经验固化为标准化流程 |
| **圆桌** (zaoren-roundtable) | 结构化多角色辩论 | 让不同skill按协议互相质询 |

> 每个预设都不只是语气模板。投资人内置估值判断框架，导师内置论文评审决策树。语气是赠品，判断才是正餐。

## 实际使用示例

### 造一个导师，帮你审论文

```
你：[上传5封导师的审稿邮件]

系统：已提取导师的审稿决策树

你：用我导师的视角看看这篇投稿

系统：
- contribution 不够清晰，第三段才点题
- baseline 选了太弱的对比方法，显得提升很大但实际不公平
- 图2 的 legend 太小，看不清
- novelty 属于 incremental improvement，建议 major revision
```

### 造一个行业，评估投资机会

```
你：[上传10份储能行业研报、3个案例]

系统：已构建储能行业评估框架

你：评估这家储能公司

系统：
- 技术没问题，但并网资源是短板
- 创始人有电网背景，这是核心优势
- 但商业化进度落后竞争对手6个月
- 建议：等Q3首批并网项目落地后再评估
```

### 启动圆桌，多方辩论

```
你：让投资人、行业专家、法务讨论这个并购案

投资人：估值合理，但整合风险高...
行业专家：标的技术在行业内排前三，但客户集中度太高...
法务：知识产权有瑕疵，需要尽调...

--- 交叉质询 ---

投资人问法务：知识产权瑕疵对估值影响多大？
法务：如果核心专利有争议，估值应该打7折...
行业专家：其实技术本身不难复制，关键在人...
```

## 使用之前，你要知道

这个工具像模具——帮你把经验固化成可复用的形状，但用模具造出来的东西好不好，还得看原材料。

**第一，素材决定质量。** 你给的是聊天记录，造出来的就是语气像；给的是工作文档和决策邮件，造出来的才能做专业判断。Garbage in, garbage out。

**第二，不能替代真人。** 造出来的导师能帮你预审论文，但最终签字负责的还是真人导师。造出来的投资人能帮你分析项目，但投不投、投多少，决定权在你。

**第三，有边界。** 适合用来：分析判断提效、预审预演、多方视角碰撞。不适合用来：骚扰、冒充、欺骗、替代真人做最终决策。

**一句话：工具帮你思考和准备，但责任在你，真人不可替代。**

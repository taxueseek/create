## 项目简介

Create 系列技能是一个认知系统工厂，从任意素材中提取可复用的判断逻辑。输入可以是：人物（聊天记录/邮件/简历）、职业（工作流/案例/规范）、书籍（思想体系/方法论）、行业（研报/数据/政策）、项目（文档/决策记录/复盘）。

输出：可安装的 skill，继承原对象的认知框架。不是复刻表面，是移植底层逻辑。不是复制过去，是创造可应对新问题的工具。

## 包含的技能

| 技能名称 | 描述 | 触发关键词 |
|---------|------|------------|
| **create** | 认知系统工厂 — 从任意素材中提取可复用的判断逻辑 | 「帮我造一个...」「提取判断逻辑」 |
| **create-boss** | 老板视角 — 输入工作汇报、项目方案，输出老板的真实判断 | 「老板会怎么看」「怎么向老板汇报」 |
| **create-client** | 甲方视角 — 输入提案、方案，输出甲方的真实顾虑 | 「甲方说感觉不对」「怎么回复甲方砍价」 |
| **create-investor** | 投资人视角 — 输入BP、创业想法，输出投资人的真实判断 | 「我的BP能不能融到钱」「模拟投资人拷问」 |
| **create-mentor** | 导师视角 — 输入研究计划、论文，输出导师的评估逻辑 | 「导师会怎么push我」「用导师的标准审我的稿」 |
| **create-parents** | 父母视角 — 输入人生决定，输出父母的真实顾虑 | 「怎么跟爸妈说我要辞职」「为什么他们就是不理解我」 |
| **create-reviewer** | 书评人/影评人视角 — 输入小说、剧本，输出专业评论 | 「我的小说为什么没人看」「用专业编辑的标准审稿」 |
| **create-troll** | 杠精视角 — 输入社交媒体文案，输出可能的攻击点 | 「我的文案会不会被喷」「模拟杠精找茬」 |

## 核心功能

### 1. 认知系统提取
- **人物**：从聊天记录、邮件、演讲中提取决策模式、表达风格、人际策略
- **职业**：从工作流文档、SOP、行业规范中提取标准操作程序、质量判断标准
- **智慧**：从书籍、论文、思想体系中提取核心概念、推理框架、应用场景
- **行业**：从研报、财报、政策文件中提取价值链、关键成功因素、死亡模式
- **项目**：从BP、PRD、复盘文档中提取核心假设、决策逻辑、风险地图

### 2. 通用提取框架
- **处境分析**：对象在什么约束条件下做判断
- **恐惧识别**：对象最担心什么，这种担心如何驱动行为
- **决策树提取**：面对新信息，对象如何判断
- **表达模式**：对象如何组织信息、传递判断
- **应用场景**：认知系统在什么场景下最适用

### 3. 造物圆桌
多个 skill 同时分析一个问题，让不同认知框架碰撞，产生更全面的判断。

## 使用示例

### 造人物
```
你：[上传聊天记录] 帮我造一个我的前同事张总的skill
→ 提取张总的决策模式、说话风格、判断逻辑
```

### 造职业
```
你：帮我造一个数据分析师的skill
→ 提取DA的标准工作流、判断标准、常见陷阱
```

### 造智慧
```
你：基于《孙子兵法》，造一个竞争策略skill
→ 提取孙子的核心概念、推理框架、应用场景
```

### 造行业
```
你：基于这10份半导体研报，造一个行业分析skill
→ 提取半导体行业的价值链、成功模式、死亡陷阱
```

### 造项目
```
你：基于这个创业项目的BP和复盘，造一个决策skill
→ 提取该项目的核心假设、决策逻辑、经验规则
```

### 造物圆桌
```
你：让投资人skill、导师skill、孙子兵法skill一起评估这个机会
→ 多视角碰撞，综合建议
```

## 进化机制

- **追加素材**：上传新文档 → 自动分析 → merge进对应模块
- **对话纠正**：用户说"他不会这样，他应该是..." → 写入correction层，立即生效
- **跨类型融合**：人物 + 职业 = 有具体个性的职业角色

## 伦理边界

**允许**：
- 认知学习、思维训练、职业分析
- 行业研究、项目复盘、智慧传承
- 创意碰撞、跨界融合、认知升级

**禁止**：
- 冒充具体个人进行欺诈
- 未经同意使用隐私素材
- 恶意操控、人身攻击、非法用途

## 与其他项目的区别

```
colleague-skill → 造语气（同事怎么说话）
前任.skill      → 造记忆（前任的口头禅）
career.skill    → 造清单（职业该做什么）

create.skill    → 造判断系统
                → 不只造人，造行业、造项目
                → 造出来的东西能交叉碰撞
                → 输出的不是语录，是决策树
```
